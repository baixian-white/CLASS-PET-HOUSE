# 班级宠物屋 - 测试用例 (v2.0)

本文档包含了针对最新版班级宠物屋（包含学生端、短信验证码认证、AI点评等新功能）的全面测试用例。这些用例将作为后续自动化测试（API 测试）和浏览器界面测试的基础。

## 一、 后端 API 测试（白盒自动化覆盖重点）

这里定义了针对最新重构后的系统接口的断言与测试预期。

### 1. 教师认证模块 (/api/auth)
| ID   | 测试场景 | 请求方式 | 接口资源 | 预期结果 |
|------|---------|----------|----------|----------|
| A01  | 正常教师注册 | POST | `/register` | 200, 校验验证码，消耗通过并返回 Token，并初始化默认班级。密码应密文存储。 |
| A02  | 短信获取频控 | POST | `/send-register-code` | 测试限制时间（如 60s 内重复发）应返回 429。 |
| A03  | 错误验证码注册 | POST | `/register` | 400, "验证码错误或已过期" |
| A04  | 用户名重名拦截 | POST | `/register` | 400, "用户名已存在" |
| A05  | 正常密码登录 | POST | `/login` | 200, 返回带有 userId 的 Token。 |
| A06  | 卡密激活（防重激活）| POST | `/activate` | 200, is_activated=true；再次激活返回 400。 |

### 2. 学生端与学生账号管理 (/api/student-portal & /api/students)
| ID   | 测试场景 | 请求方式 | 接口资源 | 预期结果 |
|------|---------|----------|----------|----------|
| B01  | 教师生成班级邀请码 | POST | `/api/classes/:id/invite` | 生成有效长度的邀请码返回给教师。 |
| B02  | 初始分配学生占位 | POST | `/api/students` | 添加学生后，会在 `student_accounts` 生成空数据关联。 |
| B03  | 学生校验邀请码 | POST | `/api/student-portal/invite/check` | 200, 返回班级和待绑定的学生基本信息。 |
| B04  | 学生正式注册绑定 | POST | `/api/student-portal/register` | 200, 成功设置账户名密码与绑定手机号（带短信校验）。|
| B05  | 学生重名/重复绑定 | POST | `/api/student-portal/register` | 409, 邀请码已被注册 或 用户名被占用。|
| B06  | 学生端独立登录 | POST | `/api/student-portal/login` | 200, 返回 StudentToken。 |
| B07  | 学生端获取自身信息 | GET | `/api/student-portal/me` | 200, 能查到自己的 pet_type、food_count 等。 |
| B08  | 越权操作防御 | PUT | `/api/classes/:id` | 403, 携带 StudentToken 请求教师专属接口必须被拦截。 |

### 3. 给分/核心业务逻辑边界 (/api/history)
| ID   | 测试场景 | 请求方式 | 接口资源 | 预期结果 |
|------|---------|----------|----------|----------|
| C01  | 扣分导致负数防范 | POST | `/api/history` | food_count 不得出现小于 0 的情况（向下限 0 截断）。|
| C02  | 批量加分/撤回一致性 | POST | `/api/history/revoke` | `is_revoked` 状态必须生效，同一条记录不能重复撤回。 |
| C03  | 商品扣除徽章拦截 | POST | `/api/shop/exchange` | 商品兑换若徽章不足返回 400；兑换成功必须 **实际扣减徽章**。 |
| C04  | AI点评接口测试 | POST | `/api/ai/review` | 返回 AI 生成的学生近期表现汇总字符串。 |

## 二、 前端页面流程测试（黑盒端到端场景）

用于驱动 Browser Subagent (通过模拟用户行为测试)。

### 1. 教师基础流程 (Teacher Flow)
| ID   | 测试场景 | 操作步骤 | 预期 |
|------|---------|----------|------|
| U01  | 教师完整注册与打卡 | 进注册页 -> 填参并获取短信验证码 -> 填入激活码打通卡密 -> 进主页 | 跳转 `/`(首页)，且顶部显示正确的班级名称和配置。 |
| U02  | 学生导入及分组配置 | 进设置页 -> 添加学生 -> 随机批量生成 -> 关闭设置 | 首页列表展示新增的学生卡片与初始宠物蛋。 |
| U03  | 个性化背景图设置 | 进设置 -> 更换主题色与预置背景图 -> 保存 | 整个应用的主题色及背景生效变化。 |

### 2. 积分与奖励互动流程 (Interaction Flow)
| ID   | 测试场景 | 操作步骤 | 预期 |
|------|---------|----------|------|
| U04  | 给学生加分与成长 | 点击某个学生 -> 选 +3 分项 -> 观察进度条 | 分数刷新，宠物阶段产生变化 (例如到达 10 分进化)。 |
| U05  | 随机点名工具 | 打开底部随机点名工具 -> 点击 START -> 停止 | 页面停在一个学生的卡片上。 |
| U06  | 撤回一次误操作 | 点击右上角撤回 / 快捷撤回按钮 -> 点刚加分的学生 | 提示撤回成功，该学生分数回退。 |
| U07  | 满分毕业下发徽章 | 对该学生持续加分突破100分 -> 触发召唤神龙动画 | 学生进度清空重置，其卡片左下角显示 1 个新徽章图标。 |

### 3. 学生端交互流程 (Student Portal Flow)
| ID   | 测试场景 | 操作步骤 | 预期 |
|------|---------|----------|------|
| U08  | 学生进入激活注册 | 进入 `/student/register` -> 输入老师给的邀请码 -> 填验证码 | 能够校验并成功关联自己的原始名字。 |
| U09  | 学生挑选修改宠物 | 学生登录后 -> 点击自己宠物 -> 改宠物名称和类型 | 学生界面更新，此时切换回教师端，老师看该学生也发生变化。 |
| U10  | 学生查看排行榜与商店 | 点击历史/商店页签 -> 浏览商品和班级积分榜 | 可以只读查看其它同学分数，商店若无徽章兑换会提示拦截。 |
