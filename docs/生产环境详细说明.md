# 生产环境详细说明（腾讯云 Ubuntu 22.04 + Docker）

**概览**
- 服务器：腾讯云 Ubuntu 22.04
- 项目目录：`/srv/class-pet-house/CLASS-PET-HOUSE/`
- 部署方式：Docker + docker-compose
- 运行容器：`class-pet-house`
- 监听端口：宿主机 `80` → 容器 `3000`
- 数据库：SQLite
- 数据文件：`/srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db`
- 容器内数据路径：`/data/class_pets.db`

**目录结构**
- 项目根目录：`/srv/class-pet-house/CLASS-PET-HOUSE/`
- compose 配置：`/srv/class-pet-house/CLASS-PET-HOUSE/docker-compose.yml`
- 后端环境变量：`/srv/class-pet-house/CLASS-PET-HOUSE/deploy/.env.backend`
- 数据目录（持久化）：`/srv/class-pet-house/CLASS-PET-HOUSE/data/`

**Docker 部署配置**
- compose 文件内容摘要：
  - `app` 服务
  - `container_name: class-pet-house`
  - `ports: 80:3000`
  - `env_file: ./deploy/.env.backend`
  - `volumes: ./data:/data`

**关键环境变量说明（deploy/.env.backend）**
- `NODE_ENV=production`
- `PORT=3000`
- `JWT_SECRET=...`（生产务必修改）
- `JWT_EXPIRES_IN=7d`
- `DB_STORAGE=/data/class_pets.db`
- `ADMIN_USERNAME=...`
- `ADMIN_PASSWORD=...`
- `AI_API_KEY=...`（如无需求可留空）

**启动与重启**
- 启动：
```
cd /srv/class-pet-house/CLASS-PET-HOUSE
sudo docker-compose up -d
```
- 重建并启动：
```
cd /srv/class-pet-house/CLASS-PET-HOUSE
sudo docker-compose up -d --build
```
- 停止：
```
sudo docker-compose down
```

**日志查看**
```
sudo docker-compose logs -f app
```

**数据库备份与恢复**
- 备份：
```
cp /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db \
  /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db.bak
```
- 恢复：
```
cp /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db.bak \
  /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db
```

**数据库迁移/建表（生产必手动执行）**
- 由于 `NODE_ENV=production`，服务启动时不会自动同步表结构
- 需要新增表或字段时执行：
```
sudo docker-compose exec app node db-init.js
```

**上线流程（常规）**
1. 进入目录：`/srv/class-pet-house/CLASS-PET-HOUSE/`
2. 拉取代码：
```
git fetch
# 选择目标分支
git checkout <branch>
git pull
```
3. 如有数据库结构变更，先执行：
```
sudo docker-compose exec app node db-init.js
```
4. 重建并启动：
```
sudo docker-compose up -d --build
```

**分支说明**
- 服务器本地暂无 `v2` 分支，需要先 `git fetch` 再 `git checkout v2`

**回滚流程（常规）**
1. 停止服务：
```
sudo docker-compose down
```
2. 回退代码：
```
git checkout <回滚版本>
```
3. 恢复数据库（如需）：
```
cp /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db.bak \
  /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db
```
4. 启动服务：
```
sudo docker-compose up -d --build
```

**常见问题排查**
- 页面无法访问：
  - 检查容器状态：`docker ps`
  - 检查端口占用：`ss -lntp | grep 80`
  - 检查日志：`docker-compose logs -f app`

- 登录失败或 500：
  - 检查后端日志是否报错
  - 确认数据库文件存在且可写
  - 确认 `JWT_SECRET` 与客户端一致

- 数据不生效：
  - 确认是否命中正确数据库文件
  - 确认容器挂载是否正确：
```
docker inspect class-pet-house --format '{{json .Mounts}}'
```

**安全建议**
- 修改 `JWT_SECRET`、`ADMIN_USERNAME`、`ADMIN_PASSWORD`
- 定期备份 `class_pets.db`
- 生产环境避免暴露调试日志
- AI Key 如不使用可删除
