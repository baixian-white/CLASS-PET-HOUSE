# 学生端上线与数据库迁移说明

## 适用范围
- 生产环境当前无学生端账号/登录功能
- 现有生产数据库为 SQLite 文件（例如：`生产环境数据.db`）
- v2 分支新增学生端登录与注册

## 变更点概览
- 新增表：`student_accounts`
- 新增字段：`invite_code`（邀请码，唯一），`phone`（手机号，用于判断是否已注册）

## 迁移目标
- 不破坏原有老师端/学生积分/历史数据
- 仅补齐学生端账号表结构

## 迁移步骤（推荐）
1. 备份生产数据库文件
- 将 `生产环境数据.db` 复制一份备份（例如 `生产环境数据.db.bak`）

2. 在 v2 代码中运行一次建表同步
- 确保 `DB_STORAGE` 指向生产库文件
- 然后执行 `db-init.js`（会创建缺失表）

PowerShell 示例：
```
cd backend
$env:DB_STORAGE="E:\CODE\class-pet-house-main\生产环境数据.db"
node db-init.js
```

Linux 示例：
```
cd backend
DB_STORAGE=/path/生产环境数据.db node db-init.js
```

3. 正常启动后端服务
- 如果生产环境 `NODE_ENV=production`，`server.js` 不会自动同步表结构
- 所以上线前务必执行第 2 步

## Docker 部署（可选）
如果使用 Docker：
```
docker-compose exec app node db-init.js
```

## 上线后必做事项
- 老师端在“学生账号管理”里为学生生成邀请码
- 学生使用邀请码进行注册（用户名、密码、手机号、验证码）

## 常见问题与风险
1. 未执行迁移
- 现象：学生注册/登录时报 500
- 解决：按“迁移步骤”执行一次 `db-init.js`

2. 未生成邀请码
- 现象：学生注册提示“邀请码不存在”
- 解决：老师端生成邀请码后再注册

3. 旧学生账号无法登录
- 现在必须完成“邀请码注册”流程（写入手机号），否则登录会提示“请先使用邀请码完成注册”

4. 旧 `/student/login` 链接
- 已重定向到 `/login`，避免 404

## 需要手动转移数据吗？
- 不需要
- 仅新增表结构，不会影响原有数据
