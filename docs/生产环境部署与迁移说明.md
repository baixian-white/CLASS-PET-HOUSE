# v2 生产上线与数据库迁移说明（腾讯云 Ubuntu 22.04 + Docker）

## 1. 当前生产环境确认
- 项目目录：`/srv/class-pet-house/CLASS-PET-HOUSE/`
- 容器：`class-pet-house`
- DB 挂载：`/srv/class-pet-house/CLASS-PET-HOUSE/data` → 容器内 `/data`
- DB 文件：`/srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db`
- `DB_STORAGE=/data/class_pets.db`
- `NODE_ENV=production`（不会自动同步表结构）

## 2. 迁移目标
新增学生端账号体系：
- 新表：`student_accounts`
- 关键字段：`invite_code`（唯一），`phone`（用于判断是否已完成注册）

## 3. 上线前迁移步骤（必须执行）
### 3.1 备份数据库
```bash
cp /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db \
  /srv/class-pet-house/CLASS-PET-HOUSE/data/class_pets.db.bak
```

### 3.2 执行建表/同步
> 注意：`NODE_ENV=production` 时不会自动同步，所以必须手动执行一次。

```bash
cd /srv/class-pet-house/CLASS-PET-HOUSE
docker-compose exec app node db-init.js
```

该命令只会创建缺失表，不会破坏原有数据。

## 4. 上线流程
```bash
cd /srv/class-pet-house/CLASS-PET-HOUSE

git fetch
# 切换到 v2 分支
git checkout v2
# 拉取最新代码
git pull

# 构建并重启容器
docker-compose up -d --build
```

## 5. 上线后必做
- 老师端进入「设置 → 学生账号管理」，为学生生成邀请码
- 学生使用邀请码进行注册（用户名/密码/手机号/验证码）

## 6. 登录风险评估
### 风险 1：未迁移表结构
- 现象：学生注册/登录报 500
- 解决：执行一次 `docker-compose exec app node db-init.js`

### 风险 2：旧学生账号无法登录
- 当前逻辑：学生必须完成邀请码注册（写入手机号）后才能登录
- 影响：生产环境原本没有学生端账号，不会导致历史账号问题

## 7. 是否需要手动转移数据？
**不需要。**
- 原有学生/老师/积分/历史数据会保留
- 只新增学生端账号相关表结构
